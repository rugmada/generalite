R comme calculatrice

Assurez-vous que R fonctionne. R sait faire pratiquement toutes les opérations mathématiques que vous pouvez imaginer. Alors essayons-en quelques-unes :

3 + 4 
5 * 5
# Lancez une opération

Remarquez comment R vous renvoie la réponse. La chose importante à noter ici est l’ordre des opérations que R utilise pour traiter les demandes. C’est à peu près la même chose que ce que vous avez appris en algèbre élémentaire, donc

2 + 2 * 3

n’est pas effectué de gauche à droite. De manière prévisible, les parenthèses sont la manière de dire à R de prioriser certaines opérations par rapport à d’autres, comme dans

(2 + 2) * 3

Si vous ne vous souvenez pas de vos leçons d’ordre des opérations datant d’il y a longtemps, pas de soucis. R n’en aura rien à faire si vous vous en tenez strictement aux parenthèses, et vous vous assurerez que R se comporte comme vous le souhaitez.

(2 + (2 * (3)))

Commentaires en R

Parfois, vous voudrez annoter votre code R afin qu’il soit plus facile à comprendre pour d’autres personnes (par exemple, vos instructeurs ou collaborateurs). Mais notez que vous pouvez également ajouter des commentaires à la fin du code actif :

5 / 7 # Ceci divise cinq par sept

R ignore tout ce qui suit le symbole dièse sur une ligne. Pour des commentaires sur plusieurs lignes, vous pouvez utiliser la notation suivante :

# Ce
# sont

# commentaires

# plusieurs

Notez qu’il est souvent judicieux d’annoter votre code pour expliquer ce que vous avez fait. Cela vous aidera à revenir sur votre travail plus tard et à comprendre ce que vous avez fait.

Différents types d’objets en R

R peut stocker différents types d’objets. L’une des structures de données les plus simples est un vecteur.

Les vecteurs peuvent être créés de différentes manières. Par exemple :

vec1 <- c(1, 2, 3, 4, 5)  # Crée un vecteur contenant les nombres de 1 à 5
vec2 <- c("chat", "chien", "oiseau")  # Crée un vecteur contenant des chaînes de caractères

Vous pouvez également créer des séquences à l’aide de la fonction “seq”.

vec3 <- seq(1, 10, by = 2)  # Crée une séquence de 1 à 10 par pas de 2

Vous pouvez accéder aux éléments d’un vecteur en utilisant leur indice.

vec1[3]  # Cela renvoie le troisième élément du vecteur vec1

La fonction “length” renvoie la longueur d’un vecteur.

length(vec1)  # Cela renvoie la longueur du vecteur vec1

Les vecteurs peuvent être combinés avec différentes fonctions, comme “c”, “seq”, ou “rep”.

vec4 <- c(vec1, vec3)  # Concatène les vecteurs vec1 et vec3

Les matrices sont une autre structure de données importante en R.

Vous pouvez créer une matrice à l’aide de la fonction “matrix”.

mat1 <- matrix(1:9, nrow = 3)  # Crée une matrice 3x3 avec les chiffres de 1 à 9

Vous pouvez accéder aux éléments d’une matrice en utilisant l’indice de ligne et de colonne.

mat1[2, 3]  # Cela renvoie l'élément à la deuxième ligne et troisième colonne de mat1

Les listes sont une structure de données flexible en R, qui peut contenir différents types d’objets.

Vous pouvez créer une liste en utilisant la fonction “list”.

liste1 <- list(nombres = vec1, mots = vec2, matrice = mat1)  # Crée une liste avec un vecteur, une matrice, etc.

Vous pouvez accéder aux éléments d’une liste en utilisant leur nom ou leur indice.

liste1$nombres  # Cela renvoie le vecteur "nombres" de la liste liste1
liste1[[3]]  # Cela renvoie le troisième élément de la liste liste1

Les facteurs sont utilisés pour travailler avec des données catégorielles.

Vous pouvez créer un facteur à l’aide de la fonction “factor”.

facteur1 <- factor(c("a", "b", "a", "c", "b"))  # Crée un facteur à partir d'un vecteur

Vous pouvez voir les niveaux d’un facteur à l’aide de la fonction “levels”.

levels(facteur1)  # Cela renvoie les niveaux du facteur facteur1

R possède de nombreuses autres structures de données et types d’objets, mais celles-ci sont parmi les plus courantes.

Vous êtes maintenant familier avec certains des principaux types d’objets en R ! Voyons maintenant comment utiliser certaines fonctions.

Fonctions et leur utilisation en R

R a un large éventail de fonctions intégrées pour effectuer des opérations spécifiques. Par exemple :

sqrt(16)  # Cela renvoie la racine carrée de 16

La plupart des fonctions ont des arguments, qui sont des informations supplémentaires que la fonction utilise pour effectuer son travail. Par exemple :

seq(1, 10, by = 2)  # Ici, "by" est un argument spécifiant le pas dans la séquence

Certaines fonctions ont des arguments facultatifs, alors que d’autres nécessitent des arguments obligatoires pour fonctionner.

Vous pouvez trouver de l’aide sur une fonction en utilisant la fonction “help” ou le point d’interrogation.

help(seq)  # Cela ouvre la documentation sur la fonction "seq"
?seq  # De même, cela ouvre la documentation sur la fonction "seq"

Utilisez ces fonctions intégrées à votre avantage et explorez-en de nouvelles !

Manipulation de données en R

R offre de puissants outils pour manipuler les données. Voyons quelques opérations courantes.

Filtrage de données

Vous pouvez filtrer des données en fonction de conditions spécifiques à l’aide de crochets “[”.

donnees <- data.frame(
  nom = c("Alice", "Bob", "Charlie", "David", "Eva"),
  age = c(25, 30, 22, 27, 35),
  score = c(80, 75, 90, 85, 88)
)

Supposons que nous voulons filtrer les personnes dont l’âge est supérieur à 25 ans.

filtre_age <- donnees[donnees$age > 25, ]  # Sélectionne les lignes avec un âge supérieur à 25

Modification de données Vous pouvez modifier les valeurs dans un data frame en utilisant des conditions.

Supposons que nous voulons augmenter le score de 5 points pour ceux dont l’âge est supérieur à 25.

donnees$score[donnees$age > 25] <- donnees$score[donnees$age > 25] + 5

Ajout de nouvelles colonnes Vous pouvez ajouter de nouvelles colonnes à un data frame existant.

donnees$groupe <- ifelse(donnees$age > 25, "A", "B")  # Ajoute une nouvelle colonne 'groupe' basée sur une condition

Suppression de colonnes Vous pouvez supprimer des colonnes d’un data frame en utilisant le signe moins “-”.

donnees_sans_age <- donnees[, -2]  # Supprime la colonne 'age'

Combinaison de data frames Vous pouvez combiner des data frames par lignes ou par colonnes.

donnees2 <- data.frame(
  nom = c("Frank", "Grace"),
  age = c(29, 31),
  score = c(92, 87)
)
donnees_combinees <- rbind(donnees, donnees2)  # Combine les data frames par lignes

Exploration de données

Vous pouvez obtenir des résumés statistiques ou des aperçus de vos données.

summary(donnees)  # Obtient des statistiques sommaires des données
str(donnees)  # Affiche la structure des données

Ces opérations représentent une petite partie des capacités de manipulation de données en R. Il existe de nombreuses autres fonctions et méthodes pour travailler efficacement avec des données.

Visualisation de données en R

R offre une variété de bibliothèques pour la visualisation de données. Voici un exemple avec ggplot2, l’une des bibliothèques les plus populaires.

library(ggplot2)

Création d’un graphique à barres à partir des données

graphique <- ggplot(donnees, aes(x = nom, y = score, fill = groupe)) +
  geom_bar(stat = "identity", position = "dodge") +
  labs(title = "Scores par personne", x = "Nom", y = "Score") +
  theme_minimal()

print(graphique) # Affichage du graphique

Avec ggplot2, vous pouvez créer une grande variété de graphiques personnalisables pour explorer et présenter vos données.

Rendre les choses persistantes

Jusqu’à présent, tout ce que tu as fait était transitoire ; c’était visible sur ton écran mais tu n’avais aucun moyen de ramener ces résultats lorsque tu voulais les réutiliser. Rarement voudras-tu utiliser R de cette manière. Tu voudras effectuer beaucoup d’étapes sur tes données et stocker tous les résultats de toutes ces étapes. Pour cela, tu as besoin d’assigner des valeurs à des variables dans le système R. La flèche pictographique à deux caractères, <-, est l’opérateur d’assignation en R. Essaie

x <- 3

en stockant la valeur 3 dans la variable x (sans aucun affichage). La valeur est ensuite récupérable simplement en entrant

x

En associant ce concept à l’utilité de R en tant que calculatrice, tu peux effectuer une assignation et un calcul sur la même ligne, comme dans

y <- 5 - 2

Et des variables et des nombres peuvent être utilisés ensemble dans des calculs pour créer de nouvelles variables, comme dans

z <- y + 2 * x

Souviens-toi que tu peux vérifier quelle est la valeur d’une variable à tout moment, simplement en la “calculant” par elle-même, c’est-à-dire,

z

Remarque que les noms de variables peuvent être des mots et qu’ils peuvent être aussi longs que tu le veux. Note également que l’opérateur d’assignation peut être utilisé dans l’autre sens, ->.

Bien que x, y et z soient faciles à taper, des noms plus mémorables comme poids ou total pourraient être plus utiles pour te rappeler les valeurs qu’ils contiennent.

Tu as peut-être remarqué que l’assignation se produit en dernier dans l’ordre des opérations, ce qui signifie que R ne sera pas confus si tu assignes une variable à elle-même, comme dans

z * z -> z

Lâche le clavier ! Prends un instrument d’écriture :

x <- 8
x -> y
2 -> x

Que valent x et y ?

Maintenant exécute les commandes en supprimant les “##” au début des lignes et compare tes réponses.

R est sensible à la casse, donc les variables x et X sont indépendantes. Les noms de variables ne peuvent contenir que des lettres, des chiffres et des points.

De plus, R utilise un certain nombre de noms de variables probables comme fonctions, ce qui les rend des choix confus en tant que variables. Les pires d’entre elles sont c, q, t, C, D, F, I et T. Évite-les.

Tu peux rencontrer une version synonyme de l’opérateur d’assignation <- dans certains codes, =. Bien qu’il soit un caractère plus court, il peut être confus quant à la direction de l’assignation et est souvent confondu avec l’opérateur de test-logique-d’égalité, ==, dont nous parlerons plus tard. Bien que je ne recommande pas d’utiliser l’opérateur d’assignation =, tu devrais en être conscient.

Les fonctions sont tes amies Alors que les opérations de base, telles que l’addition, sont des caractéristiques inestimables de R, le véritable cœur de R se présente sous la forme de fonctions. Les fonctions, en général, prennent des arguments et les transforment en une sortie. Cette sortie peut être un test statistique détaillé, un graphique ou une simple sommation - la chose importante à réaliser est que fondamentalement toutes les fonctions en R obéissent à des règles similaires. Une fonction en R est tout ce qui peut être spécifié par un nom suivi d’un ensemble de parenthèses - contenant les arguments de la fonction. Les arguments sont les entrées dont une fonction a besoin pour effectuer sa tâche et produire la sortie souhaitée.

Cela deviendra légèrement plus clair lorsque je discuterai de l’utilitaire d’aide ci-dessous, mais les arguments peuvent être soit requis soit avoir des valeurs par défaut s’ils ne sont pas spécifiés. Parce qu’il serait délicat de garder une trace de l’ordre de 10 entrées possibles, qui pourraient toutes être facultatives, R permet de nommer ces arguments dans la fonction. Cela rend les fonctions R faciles à écrire et à lire, après un peu d’acclimatation.

Prends la fonction log(). Elle a un argument requis, le nombre pour lequel tu veux le logarithme, évidemment. Mais elle a aussi un argument optionnel, base. Cet argument a pour valeur par défaut e s’il est omis.

Donc, tu peux déterminer le logarithme naturel de quelque chose assez facilement, comme dans,

log(10)

Pour obtenir le logarithme en base 10 de 10, tu peux écrire soit

log(10, 10) 

log(10, base=10)

“Alors, pourquoi voudrais-je jamais faire la deuxième option ?” demandes-tu. Ici, cela n’aurait pas beaucoup d’importance. Ce serait légèrement plus facile à comprendre pour un lecteur de ton code dans le second cas, mais peut-être pas assez utile pour justifier la frappe supplémentaire.

Mais à mesure que nous passons à des fonctions plus complexes, surtout les fonctions graphiques comme plot, le nombre d’arguments potentiels est énorme. Veux-tu un sous-titre pour l’axe des y ? Très probablement pas, mais lorsque tu en as besoin, c’est un argument très utile à avoir en attente. Dans ces fonctions, les deux ou trois premiers arguments seront spécifiés sans nom, puis le reste ressemblera à l’argument base=10 ci-dessus. Par exemple, presque chaque plot que tu produis aura besoin d’une étiquette sur l’axe x, disons, donc tu te retrouveras à taper xlab= souvent.

Répétition des lignes de code

As-tu essayé d’appuyer sur la flèche vers le haut ? Si ton curseur est à l’invite de commande dans la fenêtre de la Console, R rappellera chacune de tes commandes précédentes lorsque tu appuieras à plusieurs reprises sur la flèche vers le haut. Si tu travailles à partir d’un script dans R Studio, tu peux placer le curseur sur la ligne que tu veux exécuter et cliquer sur le bouton “Exécuter” en haut de la fenêtre. Sur un Mac, tu peux appuyer sur CMD-Entrée, au lieu de “Exécuter.” Pour répéter plusieurs lignes de code à partir du script, surligne les lignes et appuie sur “Exécuter” ou CMD-Entrée.

Économise-toi le temps et l’énergie de retaper en réutilisant ce que tu as déjà entré. Cela peut être particulièrement utile si tu as fait une petite erreur de frappe dans une longue commande.

Faire disparaître les choses

Ci-dessus, tu as été introduit à la procédure d’assignation en R. Plus tu as de variables assignées ou importées, plus ta mémoire deviendra encombrée. Tu peux voir quelles variables sont stockées dans la mémoire de R à tout moment en tapant*

ls()

Cela peut sembler étrange, étant donné ce que je viens de te dire sur les arguments des fonctions ci-dessus, mais il se trouve simplement que la fonction ls() a des arguments, mais ils prennent tous des valeurs par défaut lorsqu’aucun n’est spécifié.

Si tu travailles dans Rstudio, tu peux également voir les variables et fonctions définies dans ton espace de travail en cliquant sur l’onglet “Espace de travail”, qui montrera non seulement les noms de variables et de fonctions mais aussi la taille des variables définies et les arguments pour chaque fonction.

Si tu as identifié une variable spécifique ou un ensemble de variables que tu aimerais supprimer de la mémoire, tu as juste besoin de taper

rm(z)

La variable “z” peut être remplacée par n’importe quel nombre de variables, séparées par des virgules.

Alternativement, on a souvent besoin de vider la mémoire de R et de repartir à zéro - surtout parce que R va, par défaut, démarrer avec les mêmes variables en mémoire que lorsque tu as fermé le programme pour la dernière fois.

La fonction ici est un peu compliquée, mais instructive,

rm(list = ls())

Ici, tu dis à R, “Je ne vais pas te donner des noms de variables individuelles, à la place, je te donne une liste de noms de variables. Pour générer cette liste, exécute la fonction qui crée la liste de toutes les variables stockées en mémoire, ls().”

Obtenir de l’aide

Lorsque tu ne sais pas ce que fait une fonction ou quels arguments elle prend, le système d’aide intégré de R est exceptionnellement pratique. La manière la plus simple d’accéder à l’aide sur une fonction spécifique, comme log, est

?log

Mais de manière équivalente, tu peux taper

help(log)

Lorsque l’élément sur lequel tu as besoin d’aide est un opérateur, plutôt qu’une fonction standard (comme l’opérateur d’addition, +), seule la seconde de ces options fonctionnera, et tu dois inclure l’opérateur entre guillemets, comme

help("+")

Les fichiers d’aide dans R peuvent sembler intimidants. Le truc est de savoir ce que tu n’as pas besoin de savoir. Souvent, tu finiras par n’utiliser que quelques-uns des arguments. La plupart des arguments auront des valeurs par défaut standard acceptables notées par quelque chose comme nom=FALSE. Tu n’as pas besoin de comprendre chaque argument dans une fonction pour obtenir ce que tu veux.

Les descriptions peuvent être utiles, mais tu voudras également regarder les exemples. Les exemples sont généralement ciblés sur des scénarios courants et sont le plus souvent bien formulés.

Enfin, la section “voir aussi” peut être utile si tu cherches une fonction mais ne connais que le nom d’une fonction similaire.

À l’inverse, tu pourrais savoir ce que fait une fonction, mais ne pas connaître le nom de cette fonction en R. Il n’y a pas de solution incroyablement satisfaisante à cela, mais une bonne première étape est de rechercher dans la documentation d’aide intégrée le concept :

help.search("mean")

Ou, de manière équivalente,

??mean

La prochaine étape est de rechercher sur le site web de R, http://www.r-project.org, toute aide qu’il peut fournir. La manière la plus simple de le faire depuis R est de taper

RSiteSearch("mean")

Avec ces outils en main, tu devrais être prêt à résoudre la plupart de tes problèmes R toi-même !

Ce tutoriel n’aura pas toutes les fonctions et méthodes dont tu as besoin. Tu devrais d’abord te fier aux installations d’aide de R - qui sont très bonnes. Bien sûr, les instructeurs sont également heureux de t’aider en cas de difficultés, en particulier lorsque tu développes un besoin pour de nouvelles méthodes en travaillant sur tes projets.

SECTION C. Introduction aux vecteurs

Génération de vecteurs aléatoires

Il est courant en programmation de vouloir générer une liste de nombres aléatoires. R a plusieurs fonctions différentes pour cela, mais je vais te montrer rnorm(), qui génère des nombres aléatoires à partir de la distribution normale standard).

Aujourd’hui, nous utiliserons qu’un seul argument de base pour la fonction rnorm(), le nombre de nombres aléatoires que nous voulons dans notre vecteur. Disons que nous voulons 110 de ces nombres aléatoires, alors en tapant

mes.aleatoires <- rnorm(110)

cela va les générer et les stocker dans notre variable désignée.

Fonctions sur les vecteurs De nombreuses fonctions prennent un vecteur comme argument, plutôt qu’un seul nombre (ou en plus d’un seul nombre). Maintenant que tu sais comment générer des vecteurs selon les spécifications, ils peuvent être très utiles. Quelques exemples :

median(mes.aleatoires) 

mean(mes.aleatoires) 

sin(rnorm(10)) 

c(1,2,3)^3

Selon la nature de la fonction, le résultat ici est une seule valeur ou un vecteur transformé de même longueur que l’original. Les possibilités de structures de données d’entrée et de sortie sont innombrables, mais j’espère que tu réaliseras la diversité des rôles que les fonctions peuvent jouer dans tes analyses.

Tracés de base Produire une figure médiocre dans R est trivial. Avec la plupart des structures de données, tu obtiendras quelque chose en tapant simplement

plot(mes.aleatoires)

On peut se rapprocher de quelque chose d’utile en changeant l’arrangement de nos données, par exemple,

plot(sort(mes.aleatoires))

qui trie notre variable avant de la tracer, produisant une meilleure vision de la distribution des valeurs.

Ou nous pouvons changer le type de graphique ou les caractéristiques du graphique, comme dans

hist(mes.aleatoires)

pour produire un histogramme montrant la distribution des nombres que nous avons tirés de la distribution normale standard.

Dans ces derniers exemples, j’ai utilisé des pseudodonnées arbitraires ou aléatoires, au lieu de vraies données, pour t’aider à comprendre la visualisation des vecteurs. Par conséquent, étiqueter les axes dans ce cas extrême est sans pertinence. Tu devrais généralement étiqueter, avec un titre et des axes, chaque figure avec des données. Plus d’informations à ce sujet ci-dessous.

SECTION D. Visualizing Datasets

Où trouver des données

Importer des données dans R depuis ton ordinateur est assez simple, mais pour l’instant, nous n’utiliserons que les ensembles de données contenus dans les packages par défaut de R. Générer une liste de ces ensembles de données disponibles est aussi simple que de taper

data()

mais ici, R te limite un peu en ne regardant pas dans tous les packages disponibles. Il te prévient de cela en bas de sa réponse et suggère que tu devrais essayer

data(package = .packages(all.available = TRUE))

ce qui te donne une liste complète, notant de manière importante dans sa sortie à partir de quel package provient chaque ensemble de données.

Obtenir et comprendre les ensembles de données Une fois que tu as trouvé un ensemble de données que tu veux utiliser (et nous utiliserons les Données sur l’Attitude de Chatterjee-Price, ‘attitude’, comme exemple), tu le charges en mémoire en tapant

data(attitude)

Pour beaucoup de ces ensembles de données trouvés par la deuxième méthode, plus approfondie, ci-dessus, cela produira une erreur, car il ne regarde que dans le package par défaut. Contourne cela en ajoutant un argument spécifiant le package de l’ensemble de données, ici en utilisant catsM, une variante de celui vu en cours

data(catsM, package="boot")

Après avoir effectué cette étape, les données sont chargées dans la variable du même nom dans la mémoire de R, (habituellement, mais pas toujours) sous la forme d’un data frame, une autre structure de données. Mieux adaptés à l’analyse statistique, les data frames sont une alternative aux structures de données standard que tu as peut-être déjà rencontrées en programmation mathématique telles que les matrices ou les tableaux. Bien qu’ils soient incroyablement sophistiqués, je ne pourrai pas en introduire la plupart ici. Nous nous en tiendrons aux bases absolues; pour un traitement plus approfondi des data frames et d’autres structures de données dans R, consulte le site web de Travis Porco : http://www.mathepi.com/comp/index.html.

Tout d’abord, tu voudras probablement jeter un œil aux données elles-mêmes, juste pour voir ce qu’elles contiennent. Comme tu l’as appris, cela peut être fait simplement en tapant le nom de ta nouvelle variable sur une ligne seule

attitude

Le graphique résultant (le plus souvent) aura des lignes et des colonnes nommées qui correspondent aux données pour ces conditions et propriétés. Dans cet exemple, les individus sont numérotés de 1 à 30 et chacun a des réponses numériques correspondantes à différentes questions concernant leur attitude.

Tu peux également simplement visualiser le haut de l’ensemble de données (ce qui te permet également de voir les noms des colonnes) en tapant

head(attitude)

Une autre étape critique lors de la manipulation de tout ensemble de données intégré est de consulter les informations accompagnant l’ensemble de données, accessibles en tapant

help(attitude)

ou, si nécessaire, par analogie avec les problèmes de package introduits ci-dessus,

help(catsM, package="boot")

Ce texte inclura des informations clés sur les procédures et les hypothèses impliquées dans la collecte de données, en plus d’une compréhension de base des composants des données. Les exemples du fichier fournissent également des points de départ pour la visualisation ou l’analyse des données.

En plus des data frames, certains ensembles de données R incluent des collections de données plus spécialisées, telles que des séries temporelles ou des graphiques de distance. Tu sauras si tu as un tel ensemble de données à partir du fichier d’aide associé à l’ensemble de données (ou de son échec à se comporter de manière prévisible par rapport aux commandes de data-frame). N’hésite pas à utiliser le système d’aide pour trouver des façons de visualiser ces autres collections de données ! NOTE : Dans quelques jours, nous ferons un tutoriel complet sur la visualisation des données sur les maladies infectieuses.

Les figures sont produites de la même manière que ci-dessus, avec quelques nouvelles considérations.

Les analyses appropriées et significatives seront fortement dépendantes de la structure des données. Des résultats rapides, simples et modérément intéressants sont produits indépendamment-par-colonne avec des commandes simples sur l’ensemble de données en tant que tel, par exemple,

summary(attitude) 
boxplot(attitude)

Pour les data frames, les colonnes sont souvent comparées de différentes manières, donc une notation spéciale, le signe dollar, $, est réservée pour accéder aux colonnes individuelles. La syntaxe est simplement data.frame$nom.de.la.colonne. Pour voir ce que cela signifie pour R, nous pouvons taper la notation pour la première colonne de notre data frame attitude, la colonne rating

attitude$rating

Le résultat est simplement un vecteur des valeurs, ordonnées identiquement à la colonne originale. Cela signifie que tu peux invoquer n’importe quelle fonction avec laquelle tu es familier sur des vecteurs, comme dans,

hist(attitude$complaints) 
plot(sort(attitude$critical))

D’autres fonctions examineront la relation entre les colonnes, sur laquelle on peut dire beaucoup de choses.

L’exemple le plus simple est d’ajouter un second vecteur à notre fonction plot, ce qui placera les valeurs de deux colonnes sur chacun des axes.

plot(attitude$critical, attitude$privileges)

Une fois que tu as trouvé des figures utiles à produire, tu dois t’assurer qu’elles sont étiquetées de manière appropriée. Les arguments de la plupart des fonctions graphiques comprennent (entre autres) xlab, ylab et main. Voici un exemple d’un graphique finalisé.

plot(catsM$Bwt,catsM$Hwt,
     xlab="Poids corporel du chat mâle (kg)",
     ylab="Poids du cœur du chat mâle (g)",
     main="Poids corporel vs Poids du cœur pour les chats mâles de plus de 2 livres")

Toujours étiqueter tes axes (et fournir des unités, quand pertinent). R générera parfois des étiquettes par défaut, mais tu devras te demander si ces étiquettes sont suffisamment informatives (elles ne le sont généralement pas). Les figures doivent également généralement avoir des titres descriptifs.

Armé des fichiers d’aide et de cette proto-introduction à la visualisation, tu devrais commencer à voir comment tu peux raconter une histoire avec un ensemble de données.
